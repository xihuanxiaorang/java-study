### 设计模式原则

设计模式遵循 [六大原则](六大设计原则)：单一职责 (一个类和方法只做一件事)、开闭原则 (抽象架构，扩展实现)、里氏替换原则 (多态，子类可以扩展父类)、迪米特原则 (最少知道，降低耦合)、接口隔离原则 (建立单一接口)、依赖倒置原则 (细节依赖抽象、下层依赖上层)。
